# Documentation Enumerations

Purpose  
Define the standardized value lists used across documentation for Doc Type, Status, and Change Log date format.

Scope  
Only these items are standardized. Other lists in docs are examples unless explicitly marked as required.

---

## Doc Type (standard values)

Use one of the following:

- How-To
- Runbook
- Guide
- Reference
- Decision (ADR)

---

## Status (standard values)

Use one of the following:

- Draft
- Active
- Needs Review
- Deprecated

---

## Change Log date format

Use this format for all Change Log entries:

YYYY-MM-DD

Example:

2026-01-02 - *Initial creation*
